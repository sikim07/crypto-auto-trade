# 변경 이력

## v4.0 (2026-06-01) — 전략 F R:R 개선 + 과매매 방지

### 배경

2026-05-29~06-01 실전 로그 분석 (29건 매매, 누적 -9%+).

핵심 문제 5가지:
1. **비대칭 R:R** — 평균 승 +0.35% vs 평균 패 -0.82%
2. **트레일링 너무 타이트** — 0.3% 오프셋이 1분봉 노이즈(0.3~0.5%) 범위
3. **이미 오른 진입** — PROXIMITY 2%가 VWAP 대비 과도한 거리 허용
4. **VWAP 붕괴 과민** — 진입 직후 VWAP 재계산으로 미세 이탈 즉시 손절
5. **같은 종목 과매매** — PUNDIX 7회, PRL 5회 (후반부 전패)

### 신규 기능

#### VWAP 붕괴 grace (VWAP_BREACH_GRACE_SEC = 120)

- 진입 120초 내: VWAP 버퍼를 기본(0.3%)의 2배(0.6%)로 확대
- 진입 120초 후: 기본 버퍼(0.3%) 복원
- 급락 시: 확대 버퍼를 넘으면 즉시 손절 (entry breach / hard stop이 백스톱)

검토 과정:
- 대안 1 (시간 기반 완전 무시): grace 동안 가격이 빠지면 손실 확대 → 기각
- 대안 2 (조건부 grace = 채택): 버퍼 2배로 미세 이탈만 무시, 큰 이탈은 즉시 처리

#### 종목별 일일 매매 제한 (MAX_DAILY_TRADES_PER_TICKER = 2)

- KST 0시 기준 종목당 최대 2회/일
- 일일 카운터 초기화 시 함께 리셋
- LOSS_COOLDOWN(30분)과 역할 분리: 쿨다운은 손실 후만, 이 제한은 수익 매도 후 재진입도 차단

검토 과정:
- 반박 "LOSS_COOLDOWN과 중복 아닌가?" → 수익 매도 후에는 일반 쿨다운 5분만 적용. 수익→재진입→손실 패턴은 이 기능만 차단
- 반박 "2회 너무 적은가?" → 로그 분석 결과 3회차 이후 매매 전부 손실
